Uma introdução ao JavaScript

O que é JavaScript

Um script é um código fonte que não precisa de pré-processamento como, por exemplo, uma etapa de compilação, antes de ser executado. No contexto da Web, o script geralmente se refere a um código escrito em JavaScript. Este código é executado pelo navegador quando uma página é baixada ou é executado em resposta a um evento acionado, comandado, pelo usuário.

JavaScript é uma linguagem de programação para elaborar página web dinâmicas. Mais especificamente, JavaScript permite a criação de elementos complexos em páginas web, como por exemplo, atualizar dados dinamicamente, mostrar dados de modo interativo, fazer animações, plotar gráficos animados, entre outras tarefas.
JavaScript é uma linguagem interpretada e executada localmente no lado cliente de uma arquitetura cliente-servidor.

ECMAscript é a linguagem script que é a base de JavaScript. Essa linguagem script genérica foi desenvolvida pela Ecma International e é padronizada, formando um padrão JavaScript para o consórcio W3C.

No código acima podemos ver as seguintes situações:

  • O código responde a um evento na página web. Foi usado o click que é o evento provocado pelo mouse que dispara a função atualizarNome();
  • Neste exemplo um novo termo será inserido e armazenamos o nome na variável chamada nome;
  • Perceba que há uma concatenação de strings.

API

As APIs (Application Programming Interfaces, ou Interface de Programação de Aplicativos) são um recurso extra de JavaScript e estão no topo do núcleo da linguagem. Existem diversas APIs para os mais diversos fins, tais como:

  • As API DOM (Document Object Model) que permitem manipular HTML e CSS, criando, removendo e alterando etiquetas HTML e aplicando dinamicamente novos estilos para a sua página;
  • APIs de Geolocalização que recuperam informações geográficas, como é o caso da API do Google Maps;
  • APIs Canvas e WebGL permitem criar gráficos 2D e 3D animados;
  • APIs de áudio e vídeo que permitem maniputar elementos multimídia, tais como tocar música e mostrar vídeos em uma página da web, como capturar vídeos com a sua câmera, entre outras;

Código JavaScript externo a página HTML

Embora possamos deixar códigos JavaScript internos numa página HTML é mais comum invocar uma certa independência entre códigos deixando os códigos JavaScript num arquivo a parte.

Como fazer:

  1. Crie um arquivo fonte chamado jscrip.js no mesmo diretório do seu arquivo HMTL;
  2. Nesse arquivo acrescente o seguinte código fonte: 
          function criarParagrafo() {
          let para = document.createElement('p');
          para.textContent = 'Você clicou no botão!';
          document.body.appendChild(para);
        }

        const botao = document.getElementById('bot');

        botao.addEventListener('click', criarParagrafo);
  3. No arquivo HTML retire as etiquetas <script> e os códigos, substituindo pelo seguinte comandado <script src="jscript.js" defer></script>

O JavaScript, como uma linguagem interpretada e dependente das camadas anteriores (HTML e CSS) depende do carregamento prévio dos elementos que serão manipulados. Por isso é importante não apenas a ordem dos comandos, como também a ordem das etiquetas HTML e CSS.
No exemplo acima, vemos a palavra chave defer que informa ao navegador para continuar renderizando o conteúdo HTML até que a etiqueta <script> seja atingida.

Caso precisemos carregar diversos scripts Scripts atributo defer garantirá a o carregamento dos scripts na ordem em que aparecem na página e serão executados assim que o script e o conteúdo for baixado.

Veja um exemplo a seguir:

       <script defer src="script1.js"> <script>

       <script defer src="script2.js"> <script>

       <script defer src="script3.js"> <script>

Os comandos defer e async comandam o navegador a baixar e executar os scripts numa thread distinta, enquanto o resto da página está sendo baixado e disponibilizado. A execução dos scripts não bloqueia a página HTML. O comando async é usado se os seus scripts podem ser executados imediatamente, independentemente uns dos outros.

Caso os scripts precisem ser executados numa ordem particular ou a página HTML precise estar disponível antes da execução do script, use defer.
